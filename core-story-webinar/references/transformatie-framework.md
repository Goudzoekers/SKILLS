# Transformatie Framework — Alternatief Webinar-Model

Tweede webinar-model naast de Core Story. De skill kiest op basis van het aanbod.

## Wanneer Welk Model?

| Criterium | Core Story | Transformatie Framework |
|---|---|---|
| Aanbod type | 1 kernprobleem, 1 methode | Multi-module aanbod, breed publiek |
| Overtuigingsmechanisme | Perceptie-shifts via inhoud | Klantverhalen die transformatie TONEN |
| Structuur | 10-staps script met framework-uitleg | 3-5 transformatieverhalen + pitch |
| Ideaal voor | Coaching, consulting, 1-op-1 | Groepsprogramma's, jaarprogramma's, memberships |
| Tijdsduur | 55-70 min | 45-60 min |

## Kernprincipe

> "Hoe specifieker het detail, hoe harder het raakt. Specifieke details voelen UNIVERSELER dan vage beschrijvingen."

Geen methode uitleggen. Geen tips geven. Geen activatie-waarde. In plaats daarvan: echte mensen, echte transformaties, echte emotie. Het publiek concludeert ZELF dat ze dit ook willen.

## Structuur: 5 Transformaties + Pitch

### Intake-Checklist (VOOR je begint te schrijven)

1. Verzamel 5 klant-cases (minimum 3) met:
   - Naam en herkenbare context
   - Situatie VOOR (specifieke details, emotie)
   - Het kantelpunt (wat veranderde er)
   - Situatie NA (concreet resultaat + gevoel)
   - Welke module/onderdeel van het aanbod dit mogelijk maakte

2. Koppel elke case aan een module van het aanbod (subtiel, geen pitch)

3. Rangschik op emotionele impact:
   ```
   Transformatie 1: Licht — herkenning, lachen
   Transformatie 2: Stille pijn — herkenning die raakt
   Transformatie 3: Frustratie/aha — het inzicht
   Transformatie 4: Ontroering — het diepste verhaal
   Transformatie 5: Hoop — het bewijs dat het kan
   ```

### Per Transformatie: 4-Laags Structuur

```
LAAG 1: HERKENNING
"[Naam] zat in dezelfde situatie als veel van jullie. Ze [specifiek herkenbaar scenario]."
→ Het publiek denkt: "Dit ben ik"

LAAG 2: WAT NIET DOEN
"Ze had al [X] geprobeerd. [Specifieke methode] die niet werkte. [Waarom niet]."
→ Vijand benoemen zonder de persoon de schuld te geven

LAAG 3: HET AHA-MOMENT  
"Toen ontdekte ze [specifiek inzicht]. En alles veranderde."
→ De shift — NIET de methode uitleggen. Alleen het inzicht.

LAAG 4: HET RESULTAAT
"Vandaag [concreet resultaat]. [Emotioneel detail]. [Quote van de persoon]."
→ Het publiek denkt: "Dit wil ik ook"
```

### Het Filmprincipe

Behandel het webinar als een film. Elke transformatie is een scene. De emotionele boog over alle 5 samen:

```
T1: 😄 Lachen — lichte herkenning, toegankelijk verhaal
T2: 😢 Stille pijn — diepere herkenning, kwetsbaarheid
T3: 😤→💡 Frustratie → Aha — het kantelpunt, het inzicht
T4: 🥺 Ontroering — het meest emotionele verhaal
T5: 🌟 Hoop — het bewijs dat verandering mogelijk is
```

**Cruciaal:** De volgorde bepaalt de impact. Begin licht, bouw op, eindig met hoop. Niet andersom.

### Subtiele Module-Koppeling

Elke transformatie is (onuitgesproken) gekoppeld aan een module van het aanbod:

```
T1 → Module 1 (bijv. mindset/basis)
T2 → Module 2 (bijv. strategie/systeem)
T3 → Module 3 (bijv. implementatie)
T4 → Module 4 (bijv. doorbraak/verdieping)
T5 → Module 5 (bijv. schaal/vrijheid)
```

Je NOEMT de modules niet. Je laat het verhaal het werk doen. In de pitch worden de modules dan expliciet, en het publiek herkent: "Oh, DIT is wat [naam uit T3] deed."

### Wat je BEWUST NIET Doet

| Conventioneel webinar | Transformatie Framework |
|---|---|
| Methode uitleggen | Resultaten TONEN via verhalen |
| Tips en how-to geven | Inzichten delen die perspectief verschuiven |
| Framework op whiteboard | Emotionele scenes die raken |
| "In module 3 leer je..." | "Toen [naam] dit ontdekte, veranderde alles" |
| Activatie-waarde | Motivatie-waarde |

### Na Transformatie 5: De Pitch

Gebruik de 16-staps pitch uit `pitch-opbouw.md`. De bridge is nu:

"Je hebt net 5 mensen ontmoet die allemaal op een ander punt begonnen. Ze hadden één ding gemeen: ze besloten dat het anders moest. Als jij klaar bent voor hetzelfde..."

De Crossroads Close (uit `shared-kernel/references/pig-method.md`) sluit af.

## Minimum-Variant (3 transformaties)

Als de partner geen 5 goede cases heeft:

```
T1: Herkenning + licht (😄)
T2: Diepte + pijn + aha (😢→💡)
T3: Hoop + bewijs (🌟)
```

Dezelfde principes, compactere uitvoering. Bouw uit naar 5 zodra er meer cases zijn.

## Kwaliteitscheck per Transformatie

- ☐ Bevat een ECHTE naam en herkenbaar detail?
- ☐ Is de situatie-VOOR specifiek genoeg dat het publiek denkt "dit ben ik"?
- ☐ Is de vijand benoemd ZONDER de persoon te beschuldigen?
- ☐ Is het aha-moment een INZICHT, niet een methode-uitleg?
- ☐ Is het resultaat CONCREET (getal, tijdlijn, voelbaar verschil)?
- ☐ Bevat het een emotioneel detail dat je kunt VOELEN?
- ☐ Is de koppeling met het aanbod SUBTIEL (niet benoemd)?
- ☐ Past het in de emotionele boog (licht → diep → hoop)?
