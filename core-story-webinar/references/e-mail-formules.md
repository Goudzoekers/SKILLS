# E-mail Formules — Conversie-Templates

Herbruikbare frameworks voor alle e-mail types in de funnel.

## Onderwerpregel-Formules

| Type | Formule | Voorbeeld |
|---|---|---|
| Curiosity | "[Onverwacht element] + open loop" | "Die klant die 'nee' zei? Die belde gisteren terug." |
| Direct | "[Resultaat] in [tijdlijn]" | "Van 3 naar 17 klanten in 6 weken" |
| Persoonlijk | "[Voornaam], [herkenbare situatie]" | "[Voornaam], herken je dit?" |
| Urgentie | "[Deadline/schaarste] + consequentie" | "Nog 12 uur — daarna is het weg" |
| Social proof | "[Naam/getal] + resultaat" | "Waarom 847 coaches dit doen" |
| Contrarian | "[Gangbare wijsheid] is fout" | "Stop met content maken (serieus)" |
| Gift | "[Iets gratis] + specifiek" | "Een cadeautje: mijn beste ad template" |

**Regels:** Max 8 woorden. Geen hoofdletters (behalve namen). Geen uitroeptekens. Test altijd 2 varianten.

## Body-Frameworks

### Framework 1: PAS (Problem - Agitate - Solve)
```
[PROBLEEM:] Herkenbaar scenario in 1-2 zinnen
[AGITATE:] Waarom het erger wordt / wat het kost
[SOLVE:] De oplossing + CTA
```

### Framework 2: BAB (Before - After - Bridge)
```
[BEFORE:] Hoe het NU is (herkenbaar)
[AFTER:] Hoe het KAN zijn (verlangen)
[BRIDGE:] Hoe je van A naar B komt + CTA
```

### Framework 3: Story
```
[HOOK:] 1 zin die nieuwsgierig maakt
[CONTEXT:] Wie, wanneer, waar (kort)
[CONFLICT:] Het probleem/de uitdaging
[CLIMAX:] Het keerpunt
[RESULTAAT:] Wat er veranderde
[LINK:] Wat dit voor JOU betekent + CTA
```

### Framework 4: Value-First
```
[INZICHT:] 1 concreet bruikbaar inzicht
[WAAROM:] Waarom dit werkt
[BEWIJS:] Voorbeeld of data
[CTA:] "Wil je meer? [LINK]"
```

### Framework 5: Objection Crusher
```
[BEZWAAR:] "Je denkt misschien: [bezwaar]"
[VALIDATIE:] "Dat snap ik. [Klantnaam] dacht dat ook."
[BEWIJS:] Wat er gebeurde toen ze het toch deden
[CTA:] De volgende stap
```

## Technische Regels

- Plain text > HTML (hogere deliverability)
- Max 300 woorden per mail
- 1 CTA per mail (mag 2x herhaald)
- P.S. regel altijd gebruiken (wordt het meest gelezen na onderwerp)
- Afzender = persoonsnaam, niet bedrijfsnaam
- Reply-to = echt adres dat gelezen wordt
